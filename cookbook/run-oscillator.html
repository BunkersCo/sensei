Running the Oscillator Mini-App

The oscillator code computes a sum of damped, decaying, or periodic oscillators, convolved with (unnormalized) Gaussians, on either a regular grid (the default mesh is named “mesh”) or an unstructured grid (when “ucdmesh” is requested from the data adaptor). It can also seed and convect particles throughout the domain (available when the “particles” mesh is requested from the data adaptor).

When compiled without sensei support (ENABLE_SENSEI=OFF), the built-in analysis computes the sum of all voxel-wise autocorrelations for all window sizes up to the specified input (via -w option). It also finds the voxels with the k strongest autocorrelations (for every shift).

  • The simulation code is in miniapps/oscillators/oscillator.cpp.
  • The analysis interface is specified in miniapps/oscillators/analysis.h.
  • The actual analysis code is in miniapps/oscillators/analysis.cpp.
  • When sensei support is enabled (ENABLE_SENSEI=ON), miniapps/oscillators/dataadaptor.cpp is built and used to inform sensei what meshes and fields are available. For the mesh and ucdmesh meshes, a field named data is available. For the particles mesh, a field named velocity is available.
